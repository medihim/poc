01 / Executive Summary

POC를 바로 실행하지 말고, 먼저 검증 레이어를 선택합니다

이번 POC의 핵심은 앱 완성도 검증이 아니라, 일본 고객이 어떤 제안과 구조에서 실제 상담·예약 행동으로 이동하는지 검증하는 것입니다.

POC 시나리오는 고객에게 무엇을 제안할지의 문제이고, POC 레이어는 고객 행동을 어디까지 증명할지의 문제입니다.
구분정의이번 문서에서의 역할
검증 레이어고객 행동의 증거 깊이관심 → 상담 의향 → 예약 행동 → 거래 의향 → 운영 적합성
시나리오 타입고객에게 제안되는 프레임이뻐 상담 준비형, 특정 병원 상담회형, 특정 병원 직접 예약형, 다중 병원 예약형
실행 변수전환율을 좌우하는 UX 배치회원가입 위치, 앱 설치 위치, 예약금 위치, 문진 깊이, LINE 응대 방식

04 / Recommended Decision

1차 POC 추천 범위

모든 레이어를 한 번에 검증하지 않습니다. 1차 POC는 Level 1~3을 중심으로 진행하고, Level 4는 예약금 안내·결제 클릭 수준에서 예비 검증합니다.

우선순위실행안검증 레이어판단
1순위별성형외과 직접 예약형Level 1~3메인 실험
2순위이뻐 상담 준비형Level 1~2비교군
3순위이뻐 다중 병원 예약형 A-liteLevel 3 일부보조 실험
후행예약금 안내·결제 클릭Level 4 일부예비 검증
추천 정의
1차 POC는 일본 고객이 특정 병원 상담회 제안을 보고 상담 가능 시간을 선택하고 예약 신청까지 이동하는지를 검증합니다. 동시에 이뻐 일반 상담 준비형 제안과 비교하여 이뻐 브랜드 제안과 병원 직접 제안 중 어느 쪽이 더 강한 행동을 만드는지 확인합니다.

06 / Existing Playbook

기존 POC Playbook의 위치

기존 Playbook은 하위 실행 문서로 유지합니다. 다만 신규 전략서에서 선택한 검증 레이어에 맞춰 필요한 부분만 사용합니다.

역할 재정의
기존 Playbook은 “실행 아이디어의 모음”이 아니라, 선택된 레이어를 실행하기 위한 하위 실행안입니다. 따라서 이번 전략서에서 레이어와 시나리오를 결정한 뒤 기존 Playbook의 랜딩·메시지·CTA·Flow 항목을 골라 적용합니다.
